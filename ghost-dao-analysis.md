## 3. Why Frontend Treasury Values Are Broken

### Root Cause: 4 Bugs in `useTreasury.ts`

The backend `/api/treasury/state` returns raw BigInt strings with hex token addresses. The frontend doesn't convert them.

| Bug | File:Line | What Happens | Expected |
|-----|-----------|-------------|----------|
| **Raw amounts** | `useTreasury.ts:35` | Shows `"17000000"` | Should show `"17.00"` |
| **Hex symbols** | `useTreasury.ts:31` | Shows `"0xc4fb"` | Should show `"USDC"` |
| **USD always $0** | `useTreasury.ts:36,52` | `valueUsd: 0`, `totalValueUsd: 0` | Should calculate from prices |
| **Equal allocation** | `useTreasury.ts:42-48` | 33%/33%/34% (count-based) | Should be value-weighted |

### Why the Backend Chat Gets It Right But the Dashboard Doesn't

The agent tool `toolCheckBalance()` in `tools.ts:96-119` **does all this correctly**:
- Maps token addresses to symbols via `getTokenSymbol()`
- Divides by correct decimals (6 for Circle USDC, 18 for others)
- Calculates USD with hardcoded prices (`USDC: 1, WMON: 0.42, MON: 0.42`)
- Returns formatted `{ balances, totalValueUsd }`

But the dashboard endpoint `/api/treasury/state` in `treasury.ts:76-87` calls `unlinkService.getBalances()` **directly** — skipping all formatting. It returns raw token addresses and raw BigInt strings.

### The Fix (30 minutes)

**Option A (simplest):** Change the backend `/api/treasury/state` to call `toolCheckBalance()` instead of raw `getBalances()`. This reuses the formatting logic that already works.

**Option B (frontend-only):** Add decimal conversion + symbol mapping + price calculation in `useTreasury.ts`. The SYMBOL_MAP is missing entries for `0xc4fb...` (faucet USDC) and `0xea12...` (our deployed WMON).

## 6. What Both Systems Share (Current → DAO)

These components work for BOTH single-operator and multi-member DAO:

| Component | File | Why It Carries Over |
|-----------|------|-------------------|
| AI Agent Loop | `agent/core.ts` | Same tool-use loop, just needs dynamic system prompt per DAO |
| Tool Definitions | `agent/tools.ts` | `check_balance`, `get_price`, `execute_swap`, `generate_report` — universal |
| Unlink Service | `services/unlink.ts` | `sync`, `getBalances`, `executeSwap` — same API for single or multisig wallets |
| x402 Client | `services/x402.ts` | Burner-funded price feeds work regardless of DAO structure |
| Constraint Validator | `agent/validator.ts` | Layer 2 safety — just needs to read from governance instead of hardcoded |
| GhostVault Contract | `GhostVault.sol` | Core constraint logic stays. Only the owner changes (single → governance) |
| MockRouter | `MockRouter.sol` | DEX interface mock — independent of DAO structure |
| Frontend Chat | `AgentChat.tsx` + components | The star demo component — works for any user |
| Activity Feed | `ActivityFeed.tsx` | Just needs persistent backing store |
| Design System | `index.css` | Obsidian & gold theme, glass cards, animations — all reusable |

**Percentage of current code that carries forward: ~75%.** The AI agent, privacy model, and swap infrastructure are DAO-agnostic. Only the governance layer, member management, and access control need to be built from scratch.

## 7. Onboarding & UX Concepts

### Hackathon Onboarding (Minimal — 4 hours)

```
Connect Wallet (MetaMask)
    ↓
See Dashboard (balances, constraints, member list)
    ↓
Chat with Ghost Agent
    ↓
(Optional) Deposit via DepositPanel modal
```

No wizard, no settings configuration. The GhostVault is pre-deployed. Constraints are displayed as read-only cards.

### Post-Hackathon Onboarding (Full — 2 weeks)

```
1. Connect Wallet → Verify chain (auto-switch to Monad)
2. Create Treasury → Deploy GhostVault → Set initial constraints
   OR
   Join Treasury → Enter invite code → Receive viewing key
3. Configure Agent → Choose model, set system prompt additions
4. First Deposit → Token select → Amount → Approve → Deposit → Confirm
5. Save Viewing Key → Export for backup / share with auditors
6. Dashboard → Full member experience
```

### Settings Page Concept

| Section | Controls | Maps To |
|---------|----------|---------|
| **Constraints** | Max trade % slider, cooldown dropdown | `GhostVault.setConstraints()` |
| **Token Whitelist** | Toggle per token, add custom address | `GhostVault.setTokenAllowed()` |
| **Emergency** | Pause toggle | `GhostVault.setPaused()` |
| **Agent** | Model dropdown, custom instructions | Backend env / system prompt |
| **Members** | Invite link, role assignment, viewing keys | Backend + Unlink SDK |
| **Display** | Currency, theme, verbosity | Frontend localStorage |

### Privacy-First UX Patterns

1. **Viewing key gated content** — Blur dashboard for non-key-holders, unblur on verification
2. **Aggregate-only public view** — Non-members see "Treasury is active" but no amounts
3. **Equal-weight voting (hackathon)** — 1 member = 1 vote, avoids ZK proof complexity
4. **"Find the Trade" split-screen** — After any swap, show dashboard vs MonadScan side-by-side
5. **Selective disclosure for compliance** — Auditor viewing key shows aggregate, not individual
